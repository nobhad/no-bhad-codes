## üèóÔ∏è Architecture

### Design Tokens
Foundation-level design decisions encoded as variables:
- **Colors**: Primary palette, semantic colors, and theme variants
- **Typography**: Font scales, weights, and spacing
- **Spacing**: Consistent scale for layouts and components
- **Borders**: Radius values and border styles
- **Shadows**: Elevation system for depth
- **Animations**: Timing and easing functions

### Components
Reusable UI building blocks:
- **Atoms**: Basic elements (buttons, inputs, icons)
- **Molecules**: Simple combinations (form fields, cards)
- **Organisms**: Complex components (navigation, modals)

### Patterns
Higher-level layout and interaction patterns:
- **Page layouts**: Common page structures
- **Navigation patterns**: Menu and routing systems
- **Form patterns**: Data input and validation

**Scale**: Following a modular scale for consistent hierarchy
- Headings: 4xl ‚Üí 2xl for major sections
- Body: Base (1rem) for most content
- Small: sm/xs for captions and metadata

## üì± Responsive Design

### Breakpoint System
- **Mobile**: 375px and up (mobile-first)
- **Tablet**: 768px and up
- **Desktop**: 1200px and up
- **Large**: 1920px and up

### Fluid Scaling
Components scale smoothly between breakpoints using:
- CSS Grid and Flexbox
- Relative units (rem, %, vw/vh)
- Clamp() functions for fluid typography
